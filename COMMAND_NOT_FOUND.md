# 🔧 解决 "vercel: command not found" 错误

## ❌ 您遇到的错误

```bash
vicky.z@192 sat-discovery-platform % vercel --prod
zsh: command not found: vercel
```

## 📋 问题分析

**原因**: 直接运行 `vercel` 命令找不到

**解释**: Vercel CLI 没有全局安装在系统中，需要使用 `npx` 前缀来运行

---

## ✅ 立即解决方案

### 方案1: 使用 npx（✨ 推荐）

```bash
npx vercel --prod
```

**为什么推荐**:
- ✅ 无需全局安装
- ✅ 自动使用最新版本
- ✅ 避免权限问题
- ✅ 项目间版本独立
- ✅ 简单快速

---

### 方案2: 全局安装（可选）

如果您想以后直接使用 `vercel` 命令：

```bash
# 全局安装 Vercel CLI
npm install -g vercel

# 然后就可以直接使用
vercel --prod
```

**优点**:
- ✅ 命令更短
- ✅ 运行稍快
- ✅ 不需要 npx 前缀

**缺点**:
- ❌ 需要额外安装
- ❌ 可能需要管理员权限
- ❌ 需要手动更新

---

## 💡 我们推荐方案1（使用 npx）

**原因**:
1. **更简单**: 不需要额外安装步骤
2. **更安全**: 避免全局安装的权限问题
3. **更新鲜**: 总是使用最新版本的 Vercel CLI
4. **更独立**: 不同项目可以使用不同版本

## 🔍 技术解释

### 什么是 npx？

**npx** 是 Node.js 自带的包运行工具：

- ✅ 自动查找 npm 包
- ✅ 临时下载和运行
- ✅ 运行后不占用空间
- ✅ 总是使用最新版本

### 为什么需要 npx？

**没有 npx**:
```bash
vercel --prod
# ❌ 系统在 PATH 中找不到 vercel 命令
```

**使用 npx**:
```bash
npx vercel --prod
# ✅ npx 在 node_modules 或 npm 缓存中找到 vercel
# ✅ 自动运行
```

## 🎯 常见问题

### Q: 为什么之前用过 `npx vercel --prod` 现在还要用？

**A**: 因为 `vercel` 不是全局安装的，每次都需要 `npx` 前缀。

### Q: 可以不用 npx 吗？

**A**: 可以，但需要先全局安装：
```bash
npm install -g vercel
```

### Q: npx 会每次都下载吗？

**A**: 不会。首次会下载，之后会使用缓存，很快。

### Q: npx 安全吗？

**A**: 完全安全。npx 是 npm 官方工具，广泛使用。
